# AB — Game Design System

> 레퍼런스: Into the Breach · XCOM 2 · Teamfight Tactics · Hearthstone · Fire Emblem Engage
> 적용 범위: React 웹 프론트엔드 (이후 Unity 전환 시 동일 토큰 사용)
> 원칙: 다크 배경, 높은 대비, 기능 우선 — 모든 색상은 게임 정보를 전달한다

---

## 1. 디자인 원칙

| 원칙 | 설명 |
|---|---|
| **정보 최우선** | 색상, 테두리, 애니메이션은 반드시 게임 상태를 표현한다. 장식 목적의 비주얼 없음 |
| **즉시 독해** | 플레이어는 0.5초 내에 어떤 타일이 이동 가능하고, 어떤 유닛이 자신 것인지 파악해야 한다 |
| **다크 베이스** | 게임 보드는 항상 다크. UI 패널은 반투명 다크 오버레이로 보드와 분리 |
| **레이어드 정보** | 타일 기본 색 → 타일 효과 → 이동/공격 범위 → 선택 상태 순으로 레이어 |

## 3. 타일 색상 시스템

쿼터뷰 타일은 **3개의 레이어**로 구성된다:
1. **기본 타일 색** (지형 타입)
2. **타일 속성 색** (효과 오버레이)
3. **상태 색** (선택, 범위, 활성 유닛)

### 3-1. 기본 타일 (지형)

| 타입 | 기본 색 | 테두리 | 비고 |
|---|---|---|---|
| **도로** | `#1a2236` | `#1e2a42` | 밝은 회청 계열 |
| **평지** | `#182030` | `#1c2840` | 기준 타일 |
| **산악지** | `#1e1e2e` | `#2a2a3e` | 어두운 회보라 계열 |

### 3-2. 특수 타일 속성 (오버레이)

각 속성은 **배경 + 테두리 글로우** 조합으로 표현한다.

| 타일 속성 | 배경 오버레이 | 테두리/글로우 | 아이콘 색 |
|---|---|---|---|
| **불(Fire)** | `rgba(220,38,38,0.35)` | `#ef4444` | `#ff6b6b` |
| **물(Water)** | `rgba(37,99,235,0.30)` | `#3b82f6` | `#60a5fa` |
| **강(River)** | `rgba(29,78,216,0.55)` | `#1d4ed8` | `#93c5fd` (물결 패턴) |
| **산성(Acid)** | `rgba(74,222,128,0.25)` | `#22c55e` | `#86efac` |
| **전기(Electric)** | `rgba(250,204,21,0.28)` | `#eab308` | `#fde047` |
| **모래(Sand)** | `rgba(180,138,70,0.30)` | `#b4863a` | `#d4a760` |
| **모래폭풍** | `rgba(161,100,40,0.45)` | `#a16428` | `#f59e0b` (소용돌이) |
| **빙결(Ice)** | `rgba(147,210,255,0.25)` | `#93d2ff` | `#bae6fd` |

### 3-3. 타일 상태 (인터랙션 레이어)

| 상태 | 오버레이 | 테두리 | 용도 |
|---|---|---|---|
| **기본** | 없음 | 암묵적 그리드 선 | 일반 타일 |
| **호버** | `rgba(255,255,255,0.05)` | `rgba(255,255,255,0.15)` | 마우스 오버 |
| **내 유닛 위치** | `rgba(79,142,247,0.15)` | `#4f8ef7` 2px | 선택 전 내 유닛 |
| **선택된 유닛** | `rgba(245,183,49,0.20)` | `#f5b731` 2px + 글로우 | 현재 행동 중인 유닛 |
| **이동 가능** | `rgba(79,142,247,0.18)` | `#4f8ef7` 1px dashed | 이동 범위 |
| **이동 경로** | `rgba(79,142,247,0.30)` | `#4f8ef7` 2px | 경로 미리보기 |
| **공격 가능** | `rgba(240,71,71,0.18)` | `#f04747` 1px dashed | 공격 범위 |
| **공격 대상** | `rgba(240,71,71,0.35)` | `#f04747` 2px + 펄스 | 실제 타겟 |
| **현재 턴 유닛** | `rgba(245,183,49,0.12)` | `#f5b731` 2px + 글로우 | 지금 턴인 유닛 |

## 4. 유닛 토큰 (체스말) 디자인

### 4-1. 유닛 토큰 기본 구조
```
┌─────────────────┐
│  [클래스 아이콘] │  ← 32×32px 아이콘
│                 │
│  ████████████   │  ← 체력바 (하단)
└─────────────────┘
  [효과 아이콘 행]   ← 토큰 아래 8px 간격
```

- 토큰 크기: 타일의 70% (타일 64px 기준 → 토큰 44px)
- 테두리: 소유 플레이어 색으로 2px 테두리 + 1px 글로우

### 4-2. 플레이어 식별 색상

| 플레이어 | 토큰 테두리 | 글로우 |
|---|---|---|
| Player 1 | `#4f8ef7` (블루) | `rgba(79,142,247,0.4)` |
| Player 2 | `#f04747` (레드) | `rgba(240,71,71,0.4)` |
| Player 3 | `#34c972` (그린) | `rgba(52,201,114,0.4)` |
| Player 4 | `#f5b731` (골드) | `rgba(245,183,49,0.4)` |

> 4인 팀전: P1+P3(블루+그린) vs P2+P4(레드+골드) 권장

### 4-3. 체력바

- 전체 너비: 토큰 너비의 90%
- 높이: 4px
- 배경: `rgba(0,0,0,0.5)`
- 체력 100~60%: `#34c972`
- 체력 59~30%: `#f59e0b`
- 체력 29~0%: `#f04747`
- 체력바에 방어력 구분선: `rgba(255,255,255,0.3)` 수직선

### 4-4. 효과 아이콘 (상태 표시)

토큰 우측 하단 16×16px 아이콘 뱃지로 표시 (최대 4개)

| 효과 | 아이콘 색 | 배경 |
|---|---|---|
| 화염 | `#ff6b6b` | `rgba(220,38,38,0.8)` |
| 빙결 | `#93d2ff` | `rgba(147,210,255,0.8)` |
| 산성 | `#86efac` | `rgba(34,197,94,0.8)` |
| 독 | `#c084fc` | `rgba(168,85,247,0.8)` |
| 방패 활성 | `#f0f4ff` | `rgba(79,142,247,0.8)` |

## 5. HUD (게임 중 인터페이스)

### 5-1. 레이아웃
```
┌──────────────────────────────────────────────┐
│  [라운드 표시] [턴 순서 바]      [설정]       │  ← 상단 바 48px
├──────────────────────────────────────────────┤
│                                              │
│              게임 보드                        │
│           (쿼터뷰 그리드)                     │
│                                              │
├──────────────────────────────────────────────┤
│  [선택 유닛 정보]     [행동 버튼들]  [상대팀]  │  ← 하단 HUD 120px
└──────────────────────────────────────────────┘
```

### 5-2. 상단 바
- 배경: `rgba(8,12,20,0.85)` + `backdrop-filter: blur(12px)`
- 라운드: `Round 3 / 30` — `--text-primary`, 16px bold
- 현재 턴 유닛: 골드 강조 `--accent-gold`

### 5-3. 턴 순서 바 (드래프트 결과)

```
[A1] [B1] [A2] [B2] [A3] [B3]
 ▲현재 턴
```

- 각 슬롯: 32×32px 원형, 플레이어 색 테두리
- 현재 턴: 1.1배 확대 + 골드 글로우
- 완료된 턴: `opacity: 0.4`

### 5-4. 하단 HUD 패널
- 배경: `rgba(8,12,20,0.90)` + `backdrop-filter: blur(16px)`
- 높이: 120px

**선택 유닛 정보 영역 (좌측)**
- 유닛 아이콘 (48px), 이름, 클래스
- 체력/방어력 수치
- 현재 적용 중인 효과 아이콘

**행동 버튼 (중앙)**

| 버튼 | 색상 | 단축키 |
|---|---|---|
| 이동 | `--accent-primary` | M |
| 공격 | `--state-danger` | A |
| 스킬 | `#a855f7` (퍼플) | S |
| 화염 해제 | `#f97316` (오렌지) | F |
| 턴 종료 | `--surface-3` → `--accent-gold` 강조 | Space |

- 배경: `--bg-base` 전체

## 7. 타이포그래피

### 폰트 패밀리
- **UI 전반**: `Inter` (Google Fonts) — 게임 UI에서 가장 범용적으로 쓰이는 고가독성 폰트
- **숫자/수치**: `Inter` tabular numbers (`font-variant-numeric: tabular-nums`)
- **게임 타이틀/라운드**: `Inter` 800 weight — 굵은 헤드라인
- **코드/로그**: `JetBrains Mono` — 개발/디버그용

### 스케일

| 역할 | 크기 | 굵기 | 용도 |

## 8. 아이소메트릭 타일 스펙

### 기본 수치 (Into the Breach 스타일 참고)
- **타일 가로**: 64px
- **타일 세로**: 32px (가로의 50% — 표준 아이소메트릭 비율)
- **타일 높이감(두께)**: 8px (하단 측면)
- **아이소메트릭 각도**: 26.565° (2:1 비율, 픽셀아트 표준)
- **그리드**: 11 × 11 = 121 타일

### 렌더링 변환 공식 (Pixi.js 기준)
```
screen_x = (tile_col - tile_row) * (tileWidth / 2)
screen_y = (tile_col + tile_row) * (tileHeight / 2)
```

### 레이어 순서 (낮을수록 뒤)
1. 타일 바닥면 (기본 색)
2. 타일 속성 오버레이 (불/물/전기 등)
3. 타일 상태 오버레이 (이동범위/공격범위)
4. 물체 (바위, 나무)
5. 유닛 토큰
6. 효과 파티클
7. 선택/포커스 테두리
8. HUD 오버레이

## 9. 애니메이션 원칙

| 동작 | 방식 | 속도 | 레퍼런스 |
|---|---|---|---|
| 유닛 이동 | 타일 간 슬라이드 | 200ms/칸, ease-out | Into the Breach |
| 공격 | 빠른 전진 후 원위치 | 150ms forward, 100ms back | Into the Breach |
| 피격 | 빠른 흔들림(shake) | 80ms × 3회 | 범용 |
| 밀어냄 | 이동 애니메이션 | 150ms, ease-out | 범용 |
| 사망 | 페이드아웃 + 작아짐 | 300ms | 범용 |
| 빙결 | 파란 결정 오버레이 확장 | 250ms, ease-out | 범용 |
| 화염 | 타일/유닛 위 루프 파티클 | 루프 | 범용 |
| 타일 변환 | 이전 색 → 다음 색 크로스페이드 | 300ms | 범용 |
| 강 생성 | 물 타일들이 파란색으로 merge | 400ms | 범용 |
| 턴 시작 | 해당 유닛 골드 펄스 | 500ms pulse × 1 | TFT |
| 범위 표시 | 바깥→안으로 페이드인 | 150ms stagger | XCOM |

### 성능 원칙
- 유닛 이동/공격: 순차적으로 재생 (동시 재생 금지 — 읽기 어려움)
- 타일 하이라이트: CSS transition, GPU 가속 (`transform`, `opacity`만 사용)
- 파티클 효과: Pixi.js 파티클 시스템, 모바일에서는 단순화

## 10. 화면별 디자인 가이드

### 로비 / 메인 메뉴
- 배경: `--bg-base` + 미묘한 격자 패턴 (`rgba(255,255,255,0.03)`)
- 게임 로고: 중앙 상단
- 버튼: `--surface-2` 기반 카드형, 호버 시 `--accent-primary` 테두리

### 매치메이킹 / 대기실
- 플레이어 슬롯 카드: `--surface-1`, 유저 정보, 준비 상태
- 준비 완료: `--state-success` 강조

### 유닛 선택 (픽)
- 유닛 카드 그리드: 3열 × N행
- 선택된 유닛: `--accent-primary` 2px 테두리 + 배경 강조
- 스탯 비교 패널: 우측 고정

### 드래프트 (턴 순서 배치)
- 1~3 슬롯 (또는 1~6, 4인 기준)이 가로로 나열
- 슬롯에 드래그 앤 드롭 또는 클릭으로 유닛 배치
- 타임아웃 바: 최상단 진행 바

### 게임 중
- 위 HUD 섹션 참고

### 결과 화면
- 배경: 승리 시 `--bg-base` + 골드 파티클, 패배 시 다크 레드 비네트
- 승패 텍스트: 40px 800weight
- 통계 카드: `--surface-1` 카드 형태로 나열
