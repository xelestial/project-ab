## 2. 확정된 결정사항 전체

### ✅ R-01 우선권(Priority)
- 모든 유닛의 기본 우선권은 **1**
- 같은 우선권인 경우: 첫 라운드는 **랜덤** 결정, 이후 라운드마다 **번갈아 선공**
- 구현: 게임 세션에 `firstPlayerThisRound` 플래그 유지, 매 라운드 토글

### ✅ R-02 드래프트 페이즈
- **동시 지정** (양 플레이어가 독립적으로 자신의 유닛 순서를 정함)
- 상대 배치를 볼 수 없음
- 최종 턴 순서: 우선권 기준 인터리빙 → A1, B1, A2, B2, A3, B3

### ✅ R-03/04 이동+공격 순서
- **디폴트**: 이동 후 공격 (이동 없이 공격만도 가능, 이동만 하고 턴 종료도 가능)
- **유격대원 속성** 보유 시: 공격 후 이동 가능
- 공격 없이 이동만 → 턴 종료 가능
- 경유 타일에 아군/적 유닛이 있어도 이동 경로 통과 가능 (목적지는 빈 타일이어야 함)

### ✅ R-05 맵 경계 밀어냄
- 맵 경계에서 밀어냄 수신 시: **이동 없음 + 충돌 데미지도 없음**

### ✅ R-06 포격 장애물 위치
- 공격자와 대상 사이 경로 **어디든** 유닛/물체 1개 이상 존재하면 포격 가능

### ✅ R-07 + 아군/적군 구분 없음 + LOS 없음 (전면 재정의)

**이 게임은 아군/적군 구분이 없다.** 모든 유닛은 동일 규칙을 공유한다.
- 아군 유닛을 내가 공격할 수 있고, 아군의 방패 유닛이 내 관통 공격도 차단한다

**LOS(Line of Sight) 차단이 없다.** 원거리 공격은 경로 상의 유닛/물체와 관계없이 **임의의 타겟을 자유롭게 선택**할 수 있다.

**공격 범위 타입 최종 정의:**

| 타입 | 동작 | 방패 차단 여부 |
|---|---|---|
| **일반 원거리** | 타겟 1개, LOS 무관하게 자유 타겟 | 차단 안됨 (방패 유닛을 우회 가능) |
| **관통(Penetration)** | 타겟 + 그 뒤 **1칸** (타겟 2칸 뒤부터 안전) | **방패 유닛이 타겟이면** 뒤 1칸 차단 |
| **광선(Beam)** | 직선 전체 유닛/물체 전부 데미지 | **방패 유닛 위치에서 차단**, 그 뒤 전체 안전 |
| **포격(Artillery)** | 경로에 장애물 1개 이상 필요 조건, 타겟 지점 공격 | 일반 원거리와 동일 |

**방패 차단 정확한 메커니즘:**
- 방패 유닛 **자신은 데미지를 받음**
- 방패 유닛을 **직접 타겟**으로 하는 관통/광선만 차단 → 방패 뒤가 안전
- 방패 유닛을 **우회하여** (방패 유닛을 타겟하지 않고) 뒤 유닛을 직접 타겟하면 → 방패 무효
- → 방패의 전략적 의미: "나를 통해 뒤를 공격하는 것"을 막는 것이지, 뒤를 직접 타격하는 것은 막을 수 없음

**물체(바위, 나무) 기본 체력: 2**

### ✅ 드래프트 타임아웃
- **180초** 제한
- 타임아웃 시 **랜덤 지정**

### ✅ 산악지 기본 타일
- 도로/평지와 동일하게 이동 가능, **비주얼 차이만** 존재

### ✅ 타일 변환 + 유닛 효과 연동
- 타일이 변환되는 순간 그 위의 유닛에도 즉시 적용
- 예: 불 타일 → 물 타일로 변환 시, 그 위 유닛의 **화염 효과도 즉시 제거**

### ✅ R-08 (파생) 빙결 + 밀어냄 충돌 상호작용
- 밀어냄은 공격의 **부가 효과** (레퍼런스: 인투 더 브리치)
- 밀어내는 방향에 유닛/물체 있음 → **밀어냄 당한 유닛 이동 없음 + 1 데미지**
- 밀어내는 방향에 아무것도 없음 → **이동만, 데미지 없음**
- 충돌 대상이 **빙결 유닛**인 경우:
  - 빙결 유닛은 즉시 빙결 해제 (충돌 = 공격으로 간주)
  - 밀어냄 당한 유닛은 1 데미지 수령
  - 빙결 유닛 자신은 무적이 해제되기 직전 충돌이므로 **0 데미지** (빙결이 흡수)

### ✅ C-01 빙결 행동 제한
- 빙결 상태에서 **어떤 행동도 불가** (이동, 공격, 스킬, 화염 해제 모두 불가)
- AB.md 이동 불가 섹션 "이동할 수 없다" → "어떤 행동도 할 수 없다"로 **명세 수정 필요**

### ✅ C-02 빙결 해제 트리거
- 공격의 부가효과(밀어냄, 끌어옴 포함)로 충돌 발생 시 빙결 즉시 해제
- 충돌 대상이 빙결 유닛이면 빙결 해제 + 충돌 유닛만 1 데미지 (빙결 유닛 0 데미지)

### ✅ C-03 방패 속성 — 차단형
- 방패 유닛 **자신은 피해를 받음**
- 방패 유닛 **뒤의 유닛들은 관통/광선으로부터 보호됨** (차단형)
- → 포지셔닝 전략: 방패 유닛을 앞에 세워 뒤 유닛들을 관통으로부터 보호

### ✅ C-04 모래 타일 재분류
- **기본 타일**: 도로, 평지, 산악지 (기능 차이 없음, 시각적 차이만)
- **모래는 기본 타일이 아닌 특수 타일**로 재분류
- AB.md 기본 속성 목록에서 "모래" 제거 필요

### ✅ C-05 물 타일 vs 강 타일 — 완성된 규칙

| 구분 | 물 타일 | 강 타일 (물 3개 인접 시 생성) |
|---|---|---|
| 자발적 진입 | 가능 | **불가** (멈출 수 없음) |
| 통과(건너기) | 1칸당 이동력 2 소모 | **1칸당 이동력 2 소모**, 이동력 부족 시 이동 전체 불가 |
| 강 통과 중 이동력 부족 | 해당 없음 | 이동 자체 취소 (건너던 것도 원위치) |
| 밀어냄으로 진입 | 가능 | **가능** (단, 진입 순간 모든 효과+속성 즉시 상실) |
| 강에 빠진 상태 | 해당 없음 | 턴이 돌아올 때 **이동력 1** 소모로 탈출 가능 |
| 탈출 시 효과/속성 | 해당 없음 | 타일 효과 모두 제거 + 기본 속성(방패 등) **원복** |
| 탈출 후 재진입 | 해당 없음 | **불가** |

> **기존 AB.md "강 타일은 이동할 수 없다" → "강 타일에는 멈출 수 없다. 이동력 2 소모 시 통과 가능하며, 부족하면 이동 자체 취소"로 수정 필요**

**강에 빠진 상태 상세:**
- 강에 빠진 유닛은 모든 효과(화염/산성/독/빙결) + 모든 속성(방패 포함)을 **즉시 상실**
- 강에 있는 동안만 적용 (버프 속성은 강에서 빠진 상태일 때만 없는 것)
- 턴이 돌아오면 이동력 **1** 소모로 탈출 (강을 통과하는 것이 아니라 나오는 것이므로 1 소모)
- 탈출 후 기본 속성(방패 등) **원복**, 타일 효과는 제거된 채 복귀

### ✅ C-06 전기 타일 정의 (신규 추가)
- **생성**: 특정 유닛의 스킬에 의해 생성 (구현 예정)
- **효과**: 진입한 유닛은 매 턴 1 데미지
- **증폭 조건**:
  - 물 타일에 전기 속성 공격 → 데미지 2배
  - 전기 타일 위 유닛을 물 속성 스킬로 공격 → 데미지 2배
  - **단**, 스킬에 공격 속성이 명시된 경우만 발동 (메타데이터 `attackAttribute` 필드로 관리)

### ✅ 타일 속성 변환 규칙 (신규 확정)
- **마지막으로 공격한 속성이 타일 속성을 덮어씀**
- 예시:
  - 불 타일 + 물 속성 공격 → 물 타일
  - 물 타일 + 전기 속성 공격 → 전기 타일
  - 전기 타일 + 물 속성 공격 → 물 타일
- 메타데이터 스킬 스키마에 `attackAttribute: 'fire' | 'water' | 'electric' | 'acid' | 'ice' | null` 필드 추가 필요

### ✅ R-11 독(Poison) 효과
- 매 턴 1 데미지
- **제거 조건**: 물 속성 공격 수령 시, 물 타일 진입 시, 빙결 시
- 화염과의 차이: 턴 소비 자체 해제 불가, 특정 스킬로만 능동 제거 가능
- **빙결은 모든 기존 효과(화염/산성/독 포함)를 제거함** ← 확정

### ✅ R-12 전기 공격
- 전기 공격 스킬을 보유한 유닛 존재 (현재 테스트 유닛 3종과는 별개)

## 3. 확정 추가 (v4)

### ✅ NEW-A 강 탈출 후 행동
- 강 탈출(이동력 1 소모) 후 **일반 턴과 동일하게 남은 이동력으로 이동/공격 가능**
- 예: 이동력 3 → 탈출(1 소모) → 남은 이동력 2로 추가 이동 가능

### ✅ NEW-B 4인 팀전 드래프트 구조 — 완전 확정

- 4인 게임은 **2v2 팀전**
- 각 플레이어 3유닛 × 2명 = 팀당 **6유닛**, **1~6 공유 슬롯**
- 팀원이 **협의**하여 6개 슬롯에 각자 유닛을 하나씩 배치
- **선공 결정**: 양 팀 우선권 값 합산 → 높은 팀이 선공
- 동점 시: 랜덤, 이후 라운드마다 팀 단위로 번갈아감
- 턴 인터리빙: `팀A슬롯1, 팀B슬롯1, 팀A슬롯2, 팀B슬롯2, ... 팀A슬롯6, 팀B슬롯6`

### ✅ NEW-C 관통 + 방패 상호작용 확정
- 관통 공격이 방패 유닛을 타겟:
  - 방패 유닛: **데미지 받음**
  - 방패 뒤 유닛: **방패가 차단, 데미지 없음** (관통이 도달해야 할 뒤 1칸도 차단)
- 결과: 방패 유닛 대상 관통 = 방패 유닛만 피해, 모든 후방 안전

### ✅ R-10 화염 해제 확정
- **이동 + 공격 모두 포기**, 그 턴은 화염 해제만
- 빙결 중에는 화염 해제 불가 (C-01: 어떤 행동도 불가)

### ✅ 효과 적용 방식 분류 — 타일 기반 vs 유닛 기반 (최종 확정)

| 효과 | 적용 기반 | 상세 |
|---|---|---|
| **화염** | 유닛 기반 | 유닛에 달라붙어 이동해도 매 턴 1 데미지. 물/빙결/턴소비로 해제 |
| **독** | 유닛 기반 | 유닛에 달라붙어 이동해도 매 턴 1 데미지. 물/빙결/특정스킬로 해제 |
| **빙결** | 유닛 기반 | 유닛에 달라붙어 1턴 전체 행동 불가 |
| **산성** | **유닛+타일 이중** | 아래 상세 참고 |
| **전기** | 타일 기반 | 타일 위에 있는 동안만 매 턴 1 데미지. 벗어나면 즉시 중단 |
| **물** | 타일 기반 | 물 타일 위에서 화염/산성/독 즉시 제거, 전기 데미지 2배 |
| **모래폭풍** | 타일 기반 | 타일 위에 있는 동안 사거리 1 제한 |

### ✅ 산성(Acid) 효과 — 유닛+타일 이중 메커니즘 (최종 확정)

산성은 유일하게 **유닛과 타일을 동시에 오염**시키는 효과다.

**발생 경로 1: 산성 공격 스킬**
- 대상 유닛에 산성 효과 부여 (유닛 기반 → 이동해도 지속)
- 동시에, 대상 유닛이 서있는 타일이 **산성 타일로 변환**

**발생 경로 2: 산성 타일 접촉**
- 유닛이 산성 타일에 진입 → 유닛에 산성 효과 부여 (유닛 기반으로 전환 → 타일 벗어나도 지속)

**산성 효과 제거 조건** (유닛 기반):
- 물 타일 진입 시 / 물 속성 공격 수령 시 / 빙결 시

**산성 타일 제거 조건** (타일 변환 규칙):
- 물/얼음 속성 공격으로 타일 속성이 덮어씌워질 때

## 5. AB.md 수정이 필요한 오류 목록

| 항목 | 현재 서술 | 수정 내용 |
|---|---|---|
| 이동 불가 섹션 | "얼음이 된 상태에서는 1턴 동안 이동할 수 없다" | "빙결 상태에서는 1턴 동안 어떤 행동도 할 수 없다" |
| 강 타일 이동 | "강 타일은 이동할 수 없다" | "강 타일에는 멈출 수 없으며, 이동력을 소비하면 건너갈 수 있다" |
| 기본 타일 목록 | "도로, 평지, 들, 모래" | "도로, 평지, 산악지" (모래는 특수 타일로 이동) |
| 물 타일 이동 | 불명확 | "물 타일은 진입 가능, 1칸당 이동력 2 소모" 명시 |
| 전기 타일 | 정의 없음 | 전기 타일 섹션 신규 추가 |
| 타일 속성 변환 | 없음 | "마지막 공격 속성이 타일 속성을 덮어씀" 규칙 신규 추가 |

## 6. 기술 스택 권장안

### 백앤드
- **런타임**: Node.js (TypeScript)
- **WebSocket**: `ws` 라이브러리 (게임 이벤트 실시간 스트림)
- **HTTP REST**: 로비, 매치메이킹, 인증, 통계
- **데이터 저장**: Redis (게임 세션) + PostgreSQL (영구 로그/통계)

### 프론트앤드
- **React + TypeScript + Pixi.js** (쿼터뷰 렌더링, Unity 전환 시 렌더링 레이어만 교체)

### 공유 레이어
- **Zod** 메타데이터 스키마 (런타임 검증 + TypeScript 자동 추론)
- 스킬 메타데이터 스키마에 `attackAttribute: 'fire' | 'water' | 'electric' | 'acid' | 'ice' | null` 필드 추가

## 7. 구현 단계별 로드맵

### Phase 1: 코어 엔진 (백앤드)
1. 메타데이터 시스템 (DataBuilder, Zod 스키마, `attackAttribute` 포함)
2. 게임 상태 모델 (GameState, Unit, Tile, Map)
3. 이동 판정 시스템 (MovementValidator) — 물/강 타일 건너기 포함
4. 공격 판정 시스템 (AttackResolver) — LOS 차단, 관통, 방패 차단 포함
5. 효과 시스템 (EffectProcessor) — 빙결/화염/산성/독/전기/물/모래/전기 포함
6. 타일 시스템 (TileSvc) — 속성 변환, 강 생성/소멸 포함
7. 게임 루프 (GameLoop, RoundManager, TurnManager, DraftManager)
8. 유닛 테스트 전체 작성

### Phase 2: 네트워크 레이어
1. WebSocket 서버 및 이벤트 프로토콜 정의
2. 게임 세션 관리 (Redis)
3. HTTP REST API (로비, 인증, 통계)
4. 로그 시스템 (PostgreSQL)

### Phase 3: AI 플레이어
1. 규칙 기반 AI (휴리스틱)
2. AI vs AI 실행 환경
3. 통계/분석 수집 기반

### Phase 4: React 프론트앤드
1. Pixi.js 쿼터뷰 렌더링 레이어
2. WebSocket 게임 상태 동기화
3. 입력 처리 (이동, 공격, 드래프트)
4. 기본 UI (HUD, 결과 화면)

### Phase 5: 고도화
1. 강화학습 AI
2. 통계 대시보드
3. Unity 프론트앤드 교체
